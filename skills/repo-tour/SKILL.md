# Repo Tour

Use this skill when the user asks "what is this repo?", "where should I start?", or wants a fast orientation.

## Workflow

1. List top-level files and folders.
2. Identify likely entry points (CLI, app main, tests, configs).
3. Summarize the run/test commands.
4. Give a short map from user goals to relevant files.

## Output Format

- `Project purpose` (1-2 lines)
- `Where to start` (entry files)
- `How to run`
- `How to test`
- `Important paths`

## Constraints

- Prefer concrete file paths over generic advice.
- Keep the walkthrough short and actionable.
